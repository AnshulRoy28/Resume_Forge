# Adding Items to Your Library - Complete Guide

ResumeForge now has **detailed, type-specific forms** to capture all the information needed for each type of library item.

## How It Works

### Step 1: Navigate to Library
Click **"Library"** in the top navigation menu.

### Step 2: Click "Add Manually"
Click the **"Add Manually"** button in the top-right corner.

### Step 3: Select Item Type
A modal opens with a dropdown to select what you're adding:
- **Project** - GitHub repos, side projects, portfolio work
- **Work Experience** - Professional employment history
- **Education** - Degrees, universities, academic achievements
- **Skills** - Technical skills and proficiencies
- **Certification** - Professional certifications and licenses

### Step 4: Fill Type-Specific Form
The form **dynamically changes** based on your selection, showing only relevant fields.

## 📦 Adding a Project

**Fields:**
- **Project Name*** (e.g., "E-commerce Platform")
- **GitHub/Source URL** (e.g., "https://github.com/username/project")
- **Technologies Used** (comma-separated: "React, Node.js, PostgreSQL, AWS")
- **Project Description & Achievements*** (markdown format)

**Example Content:**
```markdown
Built a full-stack e-commerce platform with real-time inventory management.

• Implemented microservices architecture serving 50K+ daily users
• Reduced page load time by 60% through React optimization and CDN integration
• Integrated Stripe payment processing with 99.9% transaction success rate
• Deployed on AWS using Docker, Kubernetes, and CI/CD pipelines
```

**Tips:**
- Use bullet points starting with action verbs
- Quantify achievements with metrics
- Highlight technical challenges solved

## 💼 Adding Work Experience

**Fields:**
- **Job Title/Position*** (e.g., "Senior Software Engineer")
- **Company Name*** (e.g., "Google")
- **Location** (e.g., "San Francisco, CA")
- **Start Date*** (e.g., "Jan 2020")
- **End Date** (e.g., "Present" or "Dec 2023")
- **Technologies & Skills** (comma-separated: "Python, Django, AWS, Docker")
- **Responsibilities & Achievements*** (markdown format)
- **Company Website/LinkedIn** (optional)

**Example Content:**
```markdown
• Led development of microservices architecture serving 10M+ users with 99.99% uptime
• Reduced API response time by 40% through database optimization and caching strategies
• Mentored team of 5 junior engineers, improving code quality and delivery speed by 30%
• Architected and deployed CI/CD pipeline reducing deployment time from 2 hours to 15 minutes
• Collaborated with product team to define technical requirements for 3 major features
```

**Tips:**
- Start each bullet with a strong action verb (Led, Reduced, Architected, etc.)
- Include quantifiable metrics (percentages, numbers, time saved)
- Show impact on business/users
- Highlight leadership and collaboration

## 🎓 Adding Education

**Fields:**
- **Degree*** (e.g., "Bachelor of Science in Computer Science")
- **University/Institution*** (e.g., "Massachusetts Institute of Technology")
- **Location** (e.g., "Cambridge, MA")
- **Start Year*** (e.g., "2018")
- **Graduation Year** (e.g., "2022" or leave blank if current)
- **GPA** (e.g., "3.8/4.0" or "3.85")
- **Relevant Coursework** (comma-separated: "Data Structures, Algorithms, Machine Learning")
- **Achievements, Honors & Activities** (markdown format)
- **University Website** (optional)

**Example Content:**
```markdown
• Dean's List all 8 semesters
• President of Computer Science Club (2020-2022)
• Published research paper on distributed systems in ACM conference
• Teaching Assistant for Data Structures and Algorithms (2021)
• Recipient of Academic Excellence Scholarship ($20,000)
```

**Tips:**
- Include honors, awards, scholarships
- Mention leadership positions in clubs/organizations
- Add research, publications, or teaching experience
- List relevant projects or capstone work

## 🏆 Adding Certification

**Fields:**
- **Certification Name*** (e.g., "AWS Certified Solutions Architect - Professional")
- **Issuing Organization*** (e.g., "Amazon Web Services")
- **Date Obtained*** (e.g., "Dec 2023")
- **Expiry Date** (e.g., "Dec 2026" or leave blank if no expiry)
- **Credential ID** (e.g., "ABC123XYZ789")
- **Verification URL*** (e.g., "https://www.credly.com/badges/...")
- **Skills Covered** (comma-separated: "AWS, Cloud Architecture, Security")
- **Additional Details** (markdown format - optional)

**Example Content:**
```markdown
Scored 920/1000 (92%)

Key competencies demonstrated:
• Design and deploy scalable, highly available systems on AWS
• Implement cost optimization strategies
• Security best practices and compliance
• Migration strategies for enterprise applications
```

**Tips:**
- **Always include verification URL** - employers will check!
- Add credential ID for easy verification
- Mention score if impressive
- List key skills validated by the certification

## 🛠️ Adding Skills

**Fields:**
- **Skill Category*** (e.g., "Frontend Development", "Backend Development", "DevOps")
- **Skills*** (markdown format with proficiency levels)
- **Tags** (comma-separated for searchability)

**Example Content:**
```markdown
**Programming Languages:**
• Python - Expert (8+ years)
• JavaScript/TypeScript - Advanced (6+ years)
• Go - Intermediate (2+ years)

**Frameworks & Libraries:**
• React, Next.js - Expert

• Node.js, Express - Advanced

**Tools & Technologies:**
• Docker, Kubernetes - Advanced
• AWS (EC2, S3, Lambda, RDS) - Advanced
```

**Tips:**
- Group skills by category
- Include proficiency levels (Expert, Advanced, Intermediate)
- Add years of experience where relevant
- Be honest about skill levels

## What Happens Next?

### 1. Items Appear in Library Grid
Your items show up with:
- **Color-coded badges** (blue for experience, green for education, etc.)
- **Icons** representing the type
- **Preview** of content
- **Metadata** (company, dates, location) displayed

### 2. Resume Generation Uses All Items
When you generate a resume:
1. Paste the job description
2. AI analyzes ALL your library items (projects, experience, education, skills, certs)
3. AI scores each item for relevance
4. AI selects the most relevant items
5. AI organizes them into proper resume sections:
   - **Experience** → Experience section
   - **Education** → Education section
   - **Projects** → Projects section
   - **Skills** → Skills section
   - **Certifications** → Certifications section

### 3. Tailored to Each Job
The AI:
- Rewrites bullet points to match job description keywords
- Prioritizes relevant experience
- Highlights matching skills
- Quantifies impact where possible
- Uses ATS-friendly formatting

## Pro Tips

### For Best Results:
1. **Be Specific** - Include numbers, metrics, and concrete achievements
2. **Use Action Verbs** - Led, Built, Reduced, Increased, Architected, etc.
3. **Quantify Everything** - Users served, time saved, performance improved
4. **Keep It Updated** - Add new projects, skills, and certifications regularly
5. **Tag Thoroughly** - Good tags help AI match you to jobs better

### Common Mistakes to Avoid:
- ❌ Vague descriptions ("Worked on various projects")
- ❌ No metrics ("Improved performance")
- ❌ Missing dates or locations
- ❌ Forgetting verification links for certifications
- ❌ Not updating GPA or graduation year

### Markdown Formatting:
- Use `•` or `-` for bullet points
- Use `**bold**` for emphasis
- Use proper line breaks between sections
- Keep it clean and readable
