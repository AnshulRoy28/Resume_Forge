# Deploy to Render - Step by Step Guide 🚀

Complete guide to deploy ResumeForge to Render's free tier with persistent database.

## 🎯 Why Render?

- ✅ **Free tier** - No credit card required
- ✅ **Persistent disk** - SQLite database survives deployments
- ✅ **Auto-deploy** - Deploys on git push
- ✅ **HTTPS** - Free SSL certificate
- ✅ **Custom domain** - Add your own domain
- ✅ **Environment variables** - Secure secret management

## 📋 Prerequisites

1. GitHub account
2. Render account (free) - [Sign up here](https://render.com)
3. Your code pushed to GitHub

## 🚀 Deployment Steps

### Step 1: Push Your Code to GitHub

If you haven't already:

```bash
# Initialize git (if not already done)
git init

# Add all files
git add .

# Commit
git commit -m "Initial commit - ResumeForge"

# Create a new repository on GitHub, then:
git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO.git
git branch -M main
git push -u origin main
```

### Step 2: Sign Up for Render

1. Go to [render.com](https://render.com)
2. Click "Get Started for Free"
3. Sign up with GitHub (recommended)
4. Authorize Render to access your repositories

### Step 3: Create a New Web Service

#### Option A: Using Blueprint (Recommended - One Click!)

1. Click the "Deploy to Render" button below or go to your Render dashboard
2. Click "New" → "Blueprint"
3. Connect your GitHub repository
4. Render will automatically detect `render.yaml`
5. Click "Apply" - Done! 🎉

[![Deploy to Render](https://render.com/images/deploy-to-render-button.svg)](https://render.com/deploy)

#### Option B: Manual Setup

1. Go to [Render Dashboard](https://dashboard.render.com)
2. Click "New +" → "Web Service"
3. Connect your GitHub repository
4. Configure the service:

**Basic Settings:**
```
Name: resumeforge (or your preferred name)
Region: Oregon (or closest to you)
Branch: main
Runtime: Node
```

**Build & Deploy:**
```
Build Command: npm install && npm run reset-db
Start Command: npm start
```

**Plan:**
```
Instance Type: Free
```

### Step 4: Add Environment Variables

In the Render dashboard, go to "Environment" tab and add:

```
JWT_SECRET = <click "Generate" button>
NODE_ENV = production
PORT = 3000
```

**Important:** Click the "Generate" button next to JWT_SECRET to create a secure random value.

### Step 5: Add Persistent Disk (Critical!)

1. In your service settings, go to "Disks" tab
2. Click "Add Disk"
3. Configure:
   ```
   Name: resumeforge-data
   Mount Path: /app/data
   Size: 1 GB (free tier)
   ```
4. Click "Save"

**⚠️ Without this disk, your database will reset on every deployment!**

### Step 6: Deploy!

1. Click "Create Web Service" or "Manual Deploy"
2. Wait for deployment (usually 2-5 minutes)
3. Watch the logs for any errors
4. Once deployed, you'll get a URL like: `https://resumeforge.onrender.com`

## 🗄️ Database Management

### Backup Database

**Method 1: Using Render Shell**
```bash
# Access shell from Render dashboard
# Then run:
sqlite3 /app/data/resumeforge.db ".backup /tmp/backup.db"
# Download from Render dashboard
```

**Method 2: Using Render API**
```bash
# Install Render CLI
npm install -g @render/cli

# Login
render login

# Access shell
render shell <service-id>

# Backup
sqlite3 /app/data/resumeforge.db ".backup /tmp/backup.db"
```

### Restore Database

1. Access shell from Render dashboard
2. Upload backup file
3. Run:
   ```bash
   cp /tmp/backup.db /app/data/resumeforge.db
   ```
4. Restart service

### Reset Database

**⚠️ Warning: This deletes all data!**

1. Go to Render dashboard
2. Click "Shell" tab
3. Run:
   ```bash
   cd /app
   npm run reset-db
   ```
4. Restart service

## 🐛 Troubleshooting

### Deployment Fails

**Check Build Logs:**
1. Go to "Events" tab
2. Click on failed deployment
3. Read error messages

**Common Issues:**
- Missing dependencies: Run `npm install` locally first
- Build timeout: Optimize package.json
- Port issues: Ensure PORT env var is set

### Application Won't Start

**Check Logs:**
```
Look for:
- JWT_SECRET errors
- Database connection errors
- Port binding errors
```

**Solutions:**
- Verify JWT_SECRET is set and > 32 characters
- Check disk is mounted at `/app/data`
- Ensure PORT is set to 3000

### Database Resets on Deploy

**Problem:** Database loses data after deployment

**Solution:**
1. Verify persistent disk is added
2. Check mount path is `/app/data`
3. Verify disk size is at least 1GB
4. Check logs for database path: `📁 Using database at: /app/data/resumeforge.db`

### Slow Cold Starts

**Problem:** First request after inactivity is slow

**Explanation:** Free tier services spin down after 15 minutes of inactivity

**Solutions:**
- Upgrade to paid tier ($7/month) for always-on
- Use a service like [UptimeRobot](https://uptimerobot.com) to ping your app every 14 minutes
- Accept the cold start (usually 30-60 seconds)

### Out of Memory

**Problem:** Service crashes with OOM error

**Solution:**
- Free tier has 512MB RAM
- Optimize your code
- Upgrade to paid tier for more RAM

## 📈 Scaling

### Upgrade Instance

1. Go to "Settings" tab
2. Change "Instance Type"
3. Options:
   - Free: 512MB RAM, 0.1 CPU
   - Starter: 512MB RAM, 0.5 CPU ($7/mo)
   - Standard: 2GB RAM, 1 CPU ($25/mo)

### Horizontal Scaling

For high traffic:
1. Use Render's load balancing
2. Deploy multiple instances
3. Use external database (PostgreSQL)
4. Migrate from SQLite to PostgreSQL
